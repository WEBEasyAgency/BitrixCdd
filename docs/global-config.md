# Глобальный конфиг (global.php)

Значения по умолчанию для всех инфоблоков. Конфиг конкретного инфоблока может переопределить или дополнить любое значение.

Файл: `local/config/global.php`

```php
<?php
return [
    'sync_mode' => 'soft',
    'default_iblock_type' => 'content',
    'iblock_fields' => [
        'CODE' => [
            'IS_REQUIRED' => 'N',
            'DEFAULT_VALUE' => [
                'TRANSLITERATION' => 'Y',
                'TRANS_LEN'       => 100,
                'TRANS_CASE'      => 'L',
                'TRANS_SPACE'     => '-',
                'TRANS_OTHER'     => '-',
                'TRANS_EAT'       => 'Y',
                'USE_GOOGLE'      => 'N',
            ],
        ],
    ],
];
```

## Ключи

| Ключ | Что делает | По умолчанию |
|------|------------|--------------|
| `sync_mode` | Режим синхронизации для всех инфоблоков (см. [configuration.md](configuration.md#sync_mode)) | `'soft'` |
| `default_iblock_type` | Тип инфоблока по умолчанию. Подставляется если тип не указан в конфиге инфоблока. Если тип не существует в БД -- создаётся автоматически | -- |
| `iblock_fields` | Настройки стандартных полей инфоблока (см. ниже) | -- |

---

## iblock_fields

Настраивает стандартные поля инфоблока: обязательность, значения по умолчанию, автогенерация символьного кода.

Не путать с `properties`:

- `iblock_fields` -- встроенные поля Битрикс (CODE, NAME, ACTIVE и т.д.)
- `properties` -- пользовательские свойства (AUTHOR, PRICE, PHOTO и т.д.)

### Формат

```php
'iblock_fields' => [
    'FIELD_NAME' => [
        'IS_REQUIRED'   => 'Y',         // 'Y' или 'N'
        'DEFAULT_VALUE' => mixed,        // Формат зависит от поля
    ],
],
```

### Доступные поля

| Поле | Для чего |
|------|----------|
| `CODE` | Символьный код элемента |
| `ACTIVE` | Активность |
| `ACTIVE_FROM` | Дата начала активности |
| `ACTIVE_TO` | Дата окончания активности |
| `SORT` | Сортировка |
| `PREVIEW_TEXT` | Текст анонса |
| `PREVIEW_PICTURE` | Картинка анонса |
| `DETAIL_TEXT` | Детальный текст |
| `DETAIL_PICTURE` | Детальная картинка |
| `SECTION_CODE` | Символьный код раздела |

### Когда применяется

Всегда -- при каждой регистрации инфоблока, независимо от `sync_mode` и `version`.

### Где задавать

- **В `global.php`** -- чтобы не дублировать одинаковые настройки в каждом инфоблоке. Типичный случай: транслитерация CODE.
- **В конфиге инфоблока** -- когда конкретному инфоблоку нужно другое поведение. Локальные значения дополняют глобальные, а не заменяют целиком.

```php
// global.php -- CODE необязателен, транслитерация включена
'iblock_fields' => [
    'CODE' => [
        'IS_REQUIRED' => 'N',
        'DEFAULT_VALUE' => ['TRANSLITERATION' => 'Y', ...],
    ],
],

// iblocks/catalog.php -- для каталога CODE обязателен
'iblock_fields' => [
    'CODE' => ['IS_REQUIRED' => 'Y'],
    // транслитерация из global сохранится
],
```

## Транслитерация CODE

Автоматическая генерация `CODE` из `NAME` при создании элемента в админке.

```php
'CODE' => [
    'DEFAULT_VALUE' => [
        'TRANSLITERATION' => 'Y',   // Включить автогенерацию
        'TRANS_LEN'       => 100,    // Максимальная длина
        'TRANS_CASE'      => 'L',    // 'L' = нижний регистр, 'U' = верхний
        'TRANS_SPACE'     => '-',    // Замена пробелов
        'TRANS_OTHER'     => '-',    // Замена спецсимволов
        'TRANS_EAT'       => 'Y',    // Удалять символы с диакритикой
        'USE_GOOGLE'      => 'N',    // Транслитерация через Google
    ],
],
```
