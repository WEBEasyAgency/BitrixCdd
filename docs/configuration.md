# Конфигурация

## Структура конфигов

```
local/config/
├── global.php              # Глобальные настройки (мержатся во все конфиги)
├── iblock_types/           # Типы инфоблоков (один файл = один тип)
│   └── content.php
└── iblocks/                # Инфоблоки (один файл = один инфоблок)
    └── news.php
```

Путь к конфигам можно переопределить:

```php
$app->initialize([
    'config_dir' => $_SERVER['DOCUMENT_ROOT'] . '/custom/config',
]);
```

## Глобальный конфиг (global.php)

```php
<?php
return [
    'sync_mode' => 'soft',  // Режим синхронизации (см. таблицу ниже)

    // Настройки полей для CIBlock::SetFields (применяются ко ВСЕМ инфоблокам)
    'iblock_fields' => [
        'CODE' => [
            'IS_REQUIRED' => 'N',
            'DEFAULT_VALUE' => [
                'TRANSLITERATION' => 'Y',
                'TRANS_LEN'       => 100,
                'TRANS_CASE'      => 'L',
                'TRANS_SPACE'     => '-',
                'TRANS_OTHER'     => '-',
                'TRANS_EAT'       => 'Y',
                'USE_GOOGLE'      => 'N',
            ],
        ],
    ],
];
```

Все из `global.php` мержится в каждый конфиг инфоблока как значения по умолчанию. Конфиг конкретного инфоблока может переопределить любое значение, включая `sync_mode`.

---

## sync_mode

Единое свойство, управляющее синхронизацией структуры и данных.

| Режим | Структура | Демо-данные | Удаление лишнего | Когда применяется |
|----------|-----------|-------------|------------------|-------------------|
| `off` | Раз на version | Никогда | Нет | Структура без данных |
| `soft` | Раз на version | Раз (create only) | Нет | Прод после первичной заливки |
| `ensure` | Раз на version | Создать если нет, дозаполнить пустое | Нет | Гарантированная доставка данных |
| `once` | Раз на version | Раз на version (force) | Да | Разовая принудительная синхронизация |
| `danger` | Каждый хит | Каждый хит (force) | Да | Только для разработки |

Подробнее:

- **`off`** -- создает инфоблок и свойства при первом запуске (или при бампе version). Демо-данные игнорируются. Для инфоблоков, данные которых управляются только через админку.

- **`soft`** -- как `off`, но дополнительно создает элементы из `demo_data` один раз. Если элемент с таким кодом уже существует, он не трогается. При бампе version обновляется только структура (новые свойства), данные остаются как есть.

- **`ensure`** -- как `soft`, но умнее: если элемент существует, но часть свойств пустая -- дозаполняет их из `demo_data`. Уже заполненные свойства не перезаписываются. Подходит для деплоя на сервер, где инфоблок может быть в частичном состоянии.

- **`once`** -- полная принудительная синхронизация при смене version: обновляет структуру, перезаписывает данные, удаляет свойства и элементы, которых нет в конфиге. После применения -- стоп до следующего бампа version. Для случаев когда нужно привести инфоблок к конкретному состоянию.

- **`danger`** -- как `once`, но без версионирования: синхронизация происходит при каждом хите. Данные из `demo_data` перезаписывают админские правки. Только для разработки.

### Выбор режима

```
Прод, данные в админке           -> off
Прод, первичная заливка          -> soft
Прод, гарантия доставки данных   -> ensure
Миграция к конкретному состоянию -> once (бампнуть version)
Разработка                       -> danger
```

### Legacy-совместимость

Старые свойства `need_sync` и `strict` поддерживаются для обратной совместимости:

| need_sync | strict | sync_mode |
|-----------|--------|-----------|
| false | false | `off` |
| true | false | `soft` |
| false | true | `off` |
| true | true | `danger` |

Если указан `sync_mode` -- он имеет приоритет над `need_sync`/`strict`.

## Тип инфоблока

Инфоблок всегда привязан к типу. Поэтому перед созданием инфоблока нужно убедиться, что его тип существует.

Файл `local/config/iblock_types/content.php`:

```php
<?php
return [
    'id'       => 'content',
    'sections' => 'Y',
    'lang'     => [
        'ru' => [
            'NAME'         => 'Контент',
            'SECTION_NAME' => 'Разделы',
            'ELEMENT_NAME' => 'Элементы',
        ],
        'en' => [
            'NAME'         => 'Content',
            'SECTION_NAME' => 'Sections',
            'ELEMENT_NAME' => 'Elements',
        ],
    ],
];
```

## Конфиг инфоблока

```php
<?php
return [
    'version'   => 1,           // Версия конфига (подробнее в versioning.md)
    'sync_mode' => 'ensure',    // Переопределяет значение из global.php
    'priority'  => 100,         // Порядок регистрации (по умолчанию 500, меньше = раньше)

    'iblock' => [
        'code' => 'news',
        'type' => 'content',
        'name' => 'Новости',
        'sort' => 100,
    ],

    'iblock_fields' => [
        'CODE' => [
            'IS_REQUIRED' => 'Y',
        ],
    ],

    'properties' => [
        'AUTHOR' => [
            'name'     => 'Автор',
            'type'     => 'S',
            'sort'     => 100,
            'required' => true,
            'multiple' => false,
        ],
    ],

    // Ключ demo_data или demoData -- оба поддерживаются
    'demo_data' => [...],
];
```

### Типы свойств

| `type` | Описание | Примечания |
|--------|----------|------------|
| `S` | Строка | С `user_type` становится Date, DateTime, HTML |
| `N` | Число | Целое или дробное |
| `L` | Список (enum) | Требует ключ `VALUES` |
| `F` | Файл | Можно ограничить расширения через `file_type` |
| `E` | Привязка к элементу | Можно указать `link_iblock_id` (код инфоблока) |
| `G` | Привязка к разделу | Можно указать `link_iblock_id` (код инфоблока) |

### Свойства с user_type

```php
'DATE_START'   => ['name' => 'Дата начала',     'type' => 'S', 'user_type' => 'Date'],
'PUBLISHED_AT' => ['name' => 'Дата публикации', 'type' => 'S', 'user_type' => 'DateTime'],
'BODY'         => ['name' => 'Текст',           'type' => 'S', 'user_type' => 'HTML'],
```

### Свойство типа "Список" (L)

```php
'STATUS' => [
    'name'   => 'Статус',
    'type'   => 'L',
    'VALUES' => [
        ['VALUE' => 'Черновик',     'SORT' => 10, 'DEF' => 'Y'],
        ['VALUE' => 'Опубликовано', 'SORT' => 20],
        ['VALUE' => 'Архив',        'SORT' => 30],
    ],
],
```

## Программная регистрация

```php
$configService = $app->getService('iblock.config');

$configService->registerIblockFromConfig([
    'sync_mode' => 'soft',
    'iblock' => [
        'code' => 'dynamic_catalog',
        'type' => 'content',
        'name' => 'Динамический каталог',
    ],
    'properties' => [
        'PRICE' => ['name' => 'Цена', 'type' => 'N'],
    ],
]);
```

Если тип инфоблока не существует, метод попытается найти его конфиг в `iblock_types/{typeId}.php` или создаст тип с дефолтными настройками.
